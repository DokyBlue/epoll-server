# misc 模块说明

`misc` 模块提供通用基础组件：**内存管理、CRC32 校验、线程池**。这些能力被 net/logic 等核心路径高频使用。

## 目录与职责

- `nginx/misc/ngx_c_memory.cxx`：内存申请/释放封装，减少重复 `new/free` 管理。
- `nginx/misc/ngx_c_crc32.cxx`：CRC32 计算，用于包体校验。
- `nginx/misc/ngx_c_threadpool.cxx`：线程池实现（任务队列、线程唤醒、消费循环）。

## 线程池工作机制（概念）

1. 初始化固定数量工作线程。
2. 网络层在收到完整请求后将任务入队。
3. 通过条件变量/信号量唤醒线程。
4. 线程取任务并回调业务处理。
5. 停机时发出退出信号并回收线程资源。

## 在项目中的功能

- **性能**：避免每个请求都创建线程。
- **稳定性**：统一内存与任务生命周期，降低泄漏风险。
- **一致性**：CRC 提供协议级完整性校验。

## 注意事项

- 内存对象需明确“申请者/释放者”职责边界。
- 线程池任务结构建议固定头部，便于调试和统计。
- CRC 失败应尽早丢包并计数，防止污染业务线程。
