# signal 模块说明

`signal` 模块负责系统信号处理的统一入口，包括信号注册、信号到达后的行为分派。

## 目录与职责

- `nginx/signal/ngx_signal.cxx`：
  - 安装信号处理器；
  - 维护信号与动作的映射；
  - 在信号回调中设置全局状态或触发后续处理。

## 典型处理信号（视实现而定）

- `SIGTERM` / `SIGINT`：退出流程。
- `SIGHUP`：重载配置或触发维护动作。
- `SIGCHLD`：子进程状态回收（如有多进程模型）。
- `SIGPIPE`：通常忽略，避免写断链导致进程异常退出。

## 设计建议

- 信号处理函数中仅做“轻量且异步安全”的操作（置标志位）。
- 复杂清理逻辑应回到主循环上下文执行。
- 将信号行为统一日志化，提升故障可追踪性。
