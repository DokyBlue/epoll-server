# net 模块说明

`net` 模块是项目核心，负责 **监听接入、连接生命周期、epoll 事件分发、收发包处理、超时检测与发送队列**。

## 目录与职责

- `ngx_c_socket.cxx`：socket 主类框架、初始化与统一调度。
- `ngx_c_socket_accept.cxx`：新连接 accept 处理。
- `ngx_c_socket_inet.cxx`：网络地址、监听相关处理。
- `ngx_c_socket_request.cxx`：收包/发包核心流程。
- `ngx_c_socket_conn.cxx`：连接对象管理、回收与复用。
- `ngx_c_socket_time.cxx`：时间轮/超时队列及定时处理。

## 关键数据流

### 1) 连接建立

1. 监听 fd 就绪。
2. `accept` 获取 client fd。
3. 分配/复用连接对象并绑定 fd。
4. 将连接加入 epoll 读事件监听。

### 2) 收包流程（两阶段）

1. 首先读取固定长度包头。
2. 根据包头长度读取包体。
3. 组包完成后做完整性校验（含长度、CRC 等）。
4. 构造任务并投递线程池。

### 3) 发包流程

1. 业务层生成响应包并进入发送队列。
2. 发送线程尝试 `send` 直发。
3. 若内核缓冲区繁忙导致未发完：
   - 记录剩余发送偏移；
   - 注册 epoll 写事件；
   - 等待下次可写继续发送。

### 4) 超时与安全控制

- 连接心跳/超时检查。
- 恶意高频请求（洪水）识别策略。
- 异常连接延迟回收，避免并发访问悬空。

## 与其他模块协作

- 向 `logic` 模块投递“完整消息”。
- 使用 `misc` 中线程池/内存/CRC 工具。
- 依赖 `proc` 驱动事件循环。
- 通过 `app` 的日志能力记录网络异常与状态。

## 排障建议

- 重点观察连接池空闲量、发送队列长度、epoll 活跃 fd 数。
- 关注“未发完挂写事件”比例，判断是否存在背压。
- 对异常断开、包长非法、CRC 错误进行分级日志统计。
