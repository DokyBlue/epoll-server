# app 模块说明

`app` 模块负责程序级基础能力：**入口控制、配置解析、日志输出、字符串/格式化工具、进程标题设置**。

## 目录与职责

- `nginx/app/nginx.cxx`：主函数与初始化主流程。
- `nginx/app/ngx_c_conf.cxx`：配置文件加载与查询接口。
- `nginx/app/ngx_log.cxx`：日志初始化、落盘、格式化输出。
- `nginx/app/ngx_printf.cxx` / `ngx_string.cxx`：轻量工具函数。
- `nginx/app/ngx_setproctitle.cxx`：进程标题设置，便于运维观察。

## 模块对外能力

1. **配置中心能力**
   - 读取文本配置并转换为键值结构。
   - 提供按键查询（字符串、整数等）的统一接口。

2. **日志能力**
   - 支持错误级别输出、标准输出/文件输出。
   - 提供全局可复用的日志函数，供 net/logic/proc 等模块调用。

3. **运行时辅助能力**
   - 统一字符串和格式化函数，减少跨平台差异。
   - 进程标题可读化，方便 `ps/top` 排障。

## 关键初始化顺序

1. 加载配置。
2. 初始化日志系统。
3. 创建并初始化网络/线程池相关对象。
4. 进入进程循环（由 proc 模块接管）。

## 与其他模块关系

- 为 `proc` 提供启动前依赖（配置、日志）。
- 为 `net/logic/misc` 提供公共运行基础。
- 通过 `_include` 中声明的全局对象进行协同。

## 常见扩展点

- 增加新的配置项并在 `ngx_c_conf.cxx` 中提供读取逻辑。
- 增加日志维度（例如请求 ID、连接 ID）。
- 在启动阶段加入自检逻辑（端口、权限、目录可写性等）。
