# logic 模块说明

`logic` 模块承接网络层提交的完整请求，负责 **协议解释、消息码分发、业务处理、响应封包**。

## 目录与职责

- `nginx/logic/ngx_c_slogic.cxx`：业务逻辑主实现，通常包含：
  - 消息码与处理函数映射；
  - 登录/心跳等业务入口；
  - 请求合法性校验；
  - 响应消息构造与回发。

## 核心处理链

1. 从线程池消费网络层投递的任务。
2. 解析消息头与包体。
3. 根据消息码选择对应处理函数。
4. 进行业务校验（长度、权限、状态）。
5. 组装响应并调用发送接口回传。

## 模块边界

- **不直接管理 epoll/连接生命周期**（由 net 负责）。
- **不直接处理底层并发原语**（由 misc/threadpool、net 封装）。
- 专注于“业务语义正确性”和“协议级处理”。

## 扩展方式

- 新增消息类型：
  1. 定义消息码；
  2. 增加处理函数；
  3. 在分发表注册；
  4. 增加参数合法性校验与错误码回包。

## 质量建议

- 对每个消息处理入口提供统一校验模板。
- 保持“请求结构体 ↔ 响应结构体”对称，方便测试。
- 对关键业务路径增加可观测日志（不输出敏感字段）。
