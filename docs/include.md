# _include 模块说明

`_include` 是项目的**公共接口层**，定义跨模块共享的类型、常量、宏、类声明与全局对象声明。

## 主要头文件分层

1. **系统与公共基础**
   - `ngx_comm.h`：通用公共定义。
   - `ngx_macro.h`：宏常量与通用宏。
   - `ngx_func.h`：全局函数声明。

2. **全局对象与运行上下文**
   - `ngx_global.h`：全局变量、进程状态、核心对象 extern 声明。

3. **配置与工具组件接口**
   - `ngx_c_conf.h`：配置类接口。
   - `ngx_c_memory.h`：内存管理接口。
   - `ngx_c_crc32.h`：CRC32 校验接口。
   - `ngx_c_threadpool.h`：线程池接口。

4. **网络与业务接口**
   - `ngx_c_socket.h`：socket 主类、连接结构、事件处理接口。
   - `ngx_logiccomm.h`：通信协议公共结构（包头、消息头等）。
   - `ngx_c_slogic.h`：业务逻辑类接口。

5. **并发控制**
   - `ngx_c_lockmutex.h`：互斥封装。

## 设计价值

- **解耦**：各 `.cxx` 实现通过头文件契约协作。
- **可维护性**：结构体/类接口集中管理，便于定位。
- **可扩展性**：新增模块可先在 `_include` 建立接口再实现。

## 使用约束建议

- 公共头文件尽量只放声明，避免引入复杂实现逻辑。
- 对外暴露接口保持稳定，避免频繁变更影响全仓。
- 新增全局变量需审慎，优先以对象组合替代。
