# proc 模块说明

`proc` 模块负责进程级流程控制：**主循环、事件驱动、守护进程化、与信号模块协作处理生命周期**。

## 目录与职责

- `nginx/proc/ngx_process_cycle.cxx`：进程循环主流程。
- `nginx/proc/ngx_event.cxx`：事件处理组织与调度。
- `nginx/proc/ngx_daemon.cxx`：守护进程化逻辑。

## 关键流程

1. 启动后进入进程循环。
2. 调用网络层初始化监听并进入 epoll 驱动。
3. 在循环中处理 IO 事件、定时事件和退出信号。
4. 停机时按顺序释放资源（连接、线程池、日志等）。

## 生命周期管理重点

- 区分“立即退出”与“平滑退出”策略。
- 保证退出时不再接收新连接，并尽量处理完在途任务。
- 回收顺序要稳定：先停入口、再清队列、最后销毁基础设施。

## 与其他模块关系

- 调用 `signal` 安装信号处理函数。
- 驱动 `net` 的事件循环与定时检查。
- 依赖 `app` 输出运行日志。
